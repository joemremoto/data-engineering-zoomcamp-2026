Question 1. dbt Lineage and Execution
Given a dbt project with the following structure:

models/
├── staging/
│   ├── stg_green_tripdata.sql
│   └── stg_yellow_tripdata.sql
└── intermediate/
    └── int_trips_unioned.sql (depends on stg_green_tripdata & stg_yellow_tripdata)
If you run dbt run --select int_trips_unioned, what models will be built?

- only the int_trips_unioned will be built. You can add a + sign to include dependencies in the following manner:
    - --select +int_trips_unioned includes upstream deps
    - --select int_trips_unioned+ includes downstream deps
    - --select +int_trips_unioned+ includes both

What happens when you run dbt test --select fct_trips?
